# 基金自动网格买卖提醒系统 - 执行报告

## 实现概览

基于 Python 实现了完整的基金网格交易提醒系统，包含 **数据抓取**、**网格策略判断**、**通知推送**、**状态持久化** 四大核心能力。

## 项目结构

```
jijin/
├── main.py             # 主入口，支持 --now 立即执行和定时调度
├── models.py           # 数据模型（FundConfig / StrategyState / FundData）
├── data_fetcher.py     # 天天基金网数据抓取（HTTP + JSONP解析 + 3次重试）
├── strategy.py         # 网格策略核心逻辑（买入/卖出/持有信号判断）
├── notifier.py         # 通知模块（日志 + 企业微信机器人）
├── config.json         # 策略配置（支持配置初始基准价和移动网格阈值）
├── state.json          # 运行状态持久化（含 grid_center）
├── requirements.txt    # 依赖
├── grid_trading.log    # 运行日志（自动生成）
└── tests/
    └── test_strategy.py  # 14个策略单元测试
```

## 修复与升级记录

- **独立定投机制**：新增按周/按间隔天数的定投打卡提醒功能。不干涉网格买卖的仓位，触发独立专属蓝色消息通知。
- **配置文件升级为 YAML**：将 `config.json` 升级为 `config.yaml`，现在你可以直接在配置文件里使用 `#` 编写行内中文注释了，整体结构更清晰可读。
- **系统全量容器化**：编写了 `Dockerfile` 和 `docker-compose.yml`，支持一键式部署。通过 Volume 挂载实现了 `config.yaml`、`state.json` 和 `logs/` 的宿主机实时同步与持久化，极大地提升了系统运行的稳定性和便捷性。
- **项目更名与文档规范**：将项目正式命名为 **MoveGrid**，并配备了标准的 GitHub 开源文档（README.md、.gitignore 及配置模板），目前已顺利托管至 GitHub。
- **通知渠道扩展**：集成 PushPlus 推送，实现微信实时的图文提醒，并加入了“当前网格中心”展示，方便用户核对最新模型。
- **数据源迁移**：改用东方财富 `push2.eastmoney.com` HTTP 接口，兼容场内 ETF 和场外基金。
